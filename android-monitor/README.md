# Guard Monitor — мониторинг активности Android (без root)

Приложение для Android 9+ логирует изменения на телефоне (яркость, Bluetooth, Wi-Fi,
экран, громкость, геолокация) и то, какое приложение было активно в этот момент.
Отдельно сканирует, какие приложения **могут удалённо управлять/следить**
(администраторы устройства, службы спец-возможностей, опасные разрешения).

## Важное ограничение (без root)
Android **не даёт** одному приложению точно узнать, какое **другое** приложение изменило
яркость или включило Bluetooth. Поэтому виновник определяется **эвристикой** — по
приложению, которое было на переднем плане в момент события. Это не 100%: событие мог
вызвать фоновый процесс или система. Точное определение возможно только с **root**.

## Где лежит лог
```
Внутренняя память/Android/data/com.guard.monitor/files/logs/guard_log.txt
```
Подключи телефон к ПК по USB (режим «Передача файлов / MTP») и скопируй этот файл.
Формат строки: `дата время \t СОБЫТИЕ \t детали (value / foreground=пакет)`.

## Как собрать APK без Android Studio (через GitHub, бесплатно)
1. Заведи бесплатный аккаунт на github.com.
2. Создай новый репозиторий и загрузи в него **всю папку firewall** (включая
   `android-monitor/` и `.github/workflows/android-build.yml`).
   - Проще всего: на странице репозитория «Add file» → «Upload files» → перетащить папки.
3. Открой вкладку **Actions** → workflow **Build Android APK** → **Run workflow**.
4. Дождись зелёной галочки → зайди в запуск → раздел **Artifacts** →
   скачай **GuardMonitor-debug-apk** (это zip с файлом `.apk`).
5. Распакуй, перекинь `.apk` на телефон, установи (разреши «установку из этого источника»).

## Первый запуск на телефоне
1. Открой приложение **Guard Monitor**.
2. Нажми **«1) Доступ к статистике использования»** → включи для Guard Monitor
   (без этого не определяется активное приложение).
3. Нажми **«2) Разрешить уведомления»** (нужно для постоянного сервиса).
4. Нажми **«Запустить мониторинг»**.
5. Кнопка **«Сканировать сейчас»** — разовая проверка, кто может управлять устройством.

## Что в логе искать
- `BLUETOOTH state=включён` без твоего участия + `foreground=<пакет>` — кандидат.
- `ЯРКОСТЬ` меняется сама — смотри `foreground`.
- `СПЕЦ_ВОЗМОЖНОСТИ service=...` — приложения со спец-возможностями (главный вектор
  удалённого управления). Если там незнакомое приложение — подозрительно.
- `АДМИН_УСТРОЙСТВА app=...` — приложения-администраторы (могут менять политику,
  блокировать, сбрасывать).
- `ПРИЛОЖЕНИЕ_РАЗРЕШЕНИЯ app=... perms=CAMERA,RECORD_AUDIO,...` — у кого доступ к
  камере/микрофону/SMS/геолокации/поверх окон.

## Технические детали
- minSdk 28 (Android 9), targetSdk 33, Kotlin.
- Foreground-сервис + BroadcastReceiver + ContentObserver + UsageStatsManager.
- Автозапуск после перезагрузки (BootReceiver).
